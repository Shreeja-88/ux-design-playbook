# Fogg Behavior Model (FBM) 

## What is the Fogg Behavior Model?

The **Fogg Behavior Model** explains how human behavior happens.

It was created by BJ Fogg.

### Formula

```text
B = MAP
```

Where:

* **B** → Behavior
* **M** → Motivation
* **A** → Ability
* **P** → Prompt

---

# Main Idea

A behavior happens only when these 3 things occur together:

1. **Motivation** → Person wants to do it
2. **Ability** → Person can easily do it
3. **Prompt** → Something reminds or triggers them

If one element is missing, the behavior will not happen.

---

# Components of FBM

## 1. Motivation

Motivation means the desire to perform an action.

### Types of Motivation

* Personal interest
* Reward or punishment
* Social or emotional influence

### Example

* Wanting good marks motivates students to study.

---

## 2. Ability

Ability means how easy the action is.

If something is simple, people are more likely to do it.

### Ways to improve ability

* Make tasks smaller
* Provide tools/help
* Reduce complexity

### Example

* One-click login is easier than filling long forms.

---

## 3. Prompt

A prompt is a trigger that reminds users to act.

### Types of Prompts

| Prompt Type | Situation                 |
| ----------- | ------------------------- |
| Spark       | Low motivation            |
| Facilitator | Low ability               |
| Signal      | High motivation & ability |

### Examples

* App notifications
* Alarm reminders
* Email alerts

---

# Important Principles

## Principle 1

Higher motivation increases chances of behavior.

---

## Principle 2

Hard tasks reduce chances of behavior.

---

## Principle 3

Motivation and ability work together.

If motivation is low, the task must be easier.

---

## Principle 4

Without a prompt, no behavior occurs.

---

# Simple Real-Life Examples

## Example 1: Fitness App

* Motivation → Want to stay healthy
* Ability → Easy workout videos
* Prompt → Daily notification

Result → User exercises daily.

---

## Example 2: Food Delivery App

* Motivation → Hunger
* Ability → Easy ordering process
* Prompt → Discount notification

Result → User orders food.

# FBM in UX/UI Design

Designers use FBM to:

* Increase user engagement
* Build habit-forming products
* Improve app usability
* Encourage desired actions

### Common UX Uses

* Push notifications
* Reward systems
* Simple onboarding
* Gamification

---

# Advantages

* Easy to understand
* Useful in product design
* Helps improve user engagement
* Works for many behaviors

---

# Limitations

* Motivation changes over time
* Cultural differences affect behavior
* Not all human behavior is predic
table

---

# Key Takeaways

* Behavior happens when **Motivation + Ability + Prompt** occur together.
* Make actions easy for users.
* Use proper reminders/prompts.
* Reduce user effort to increase engagement.

---

# Quick Summary

```text
Behavior = Motivation + Ability + Prompt

No Prompt → No Action
Hard Task → Less Action
Easy Task + Good Motivation → More Action
```
